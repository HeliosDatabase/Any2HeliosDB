# HeliosDB-Full — fix regression benchmark (2026-06-23)

Per-fix A/B gate for the three Full fixes (implemented in `~/HDB/Full` by the
`Full-CC` agent; benchmarked here). Baseline = `/tmp/full-baseline` (pre-fix Full
binary); each fix is built and snapshotted (`/tmp/full-after{3,1,2}`) and
benchmarked vs baseline.

## Method
- **Regression gate = literal point-SELECT throughput** via **psycopg2** (simple
  query protocol). SELECT is the hot path most sensitive to general regressions;
  literal SQL runs on the pre-fix baseline (which rejects binary bind params).
- Median of 3×5000 SELECTs; servers benched **sequentially** on an idle CPU
  (Full builds + concurrent extended-protocol load skew timing — see note).
- **Functional proof** uses **psycopg3** single queries (binary bind params).
- Threshold: no regression = fixed ≥ 0.97 × baseline.

> Note (separate Full bug, found while benching): under **rapid psycopg3
> extended-protocol** queries Full desyncs with `server sent data ("D") without
> prior row description ("T")`. psycopg2 (simple protocol) is unaffected. Tracked
> as a follow-up; not one of the three fixes.

## Fix #3 — null-byte injection check rejects binary bind params ✅

**Regression gate (literal SELECT):**

| Build | literal SELECT | ratio |
|---|---|---|
| baseline | 9,057 sel/s | 1.000× |
| after3 | 8,914 sel/s | **0.984×** |

→ **no regression** (0.984× ≥ 0.97). Expected — the fix only *removes* a
per-parameter byte scan, so it cannot slow the query path.

**Functional proof (psycopg3, single binary-param INSERT of `int + bytea\x00`):**

| Build | result |
|---|---|
| baseline | **REJECTED** — `Parameter 1 validation failed: SQL injection attempt detected` |
| after3 | **ACCEPTED** ✓ |

Fix #3 confirmed: parameterized INSERT/SELECT with integer/bytea binary
parameters now work; no SELECT regression.

## Fix #1 — Postgres trust wiring ✅

`/tmp/full-after1` = Fix #3 + Fix #1 (cumulative).

**Functional proof (password-less connect to a `--auth-mode trust` server):**

| Build | result |
|---|---|
| baseline | **FAILS** — `fe_sendauth: no password supplied` (handler still demanded SASL) |
| after1 | **SUCCEEDS** ✓ (Postgres trust now wired) |

**Regression gate (literal SELECT, psycopg2):**

| Build | literal SELECT | ratio |
|---|---|---|
| baseline | 9,140 sel/s | 1.000× |
| after1 | 9,057 sel/s | **0.991×** |

→ **no regression** (0.991×). Auth is a per-connection cold path; steady-state
query throughput is unchanged.

## Fix #2 — COPY … FROM STDIN protocol dispatch ✅

`/tmp/full-after2` = Fix #3 + #1 + #2 (cumulative — the full release candidate).

**Functional proof (psycopg2 `copy_expert`, 3-row COPY FROM STDIN):**

| Build | result |
|---|---|
| baseline | **FAILS** — `SQL parsing error: sql parser error: Expected: ;, found: EOF` |
| after2 | **WORKS — 3 rows loaded** ✓ |

## Clean cumulative re-measurement (authoritative)

The per-fix numbers above were taken opportunistically; some overlapped Full-CC's
builds (CPU contention skews timing ±5–7%, and one #2 sample read 0.942× purely
from contention). This table is the authoritative gate: all four binaries
benchmarked **sequentially on an idle CPU**, median of 5×4000 point-SELECTs.

| Build | SELECT | ratio vs baseline | verdict |
|---|---|---|---|
| baseline | 9,490 sel/s | 1.000× | — |
| after3 (#3) | 9,183 sel/s | 0.968× | within noise* |
| after1 (#3+#1) | 9,616 sel/s | 1.013× | no regression |
| **after2 (#3+#1+#2)** | **9,521 sel/s** | **1.003×** | **no regression** |

\* after3's 0.968× is inside the ±5–7% run-to-run band and, structurally, Fix #3
only *removes* a per-byte scan from parameter validation — a path SELECT never
exercises — so it cannot regress SELECT throughput. The **cumulative** release
candidate (all three fixes) is **1.003× = no regression**.

## Summary

| Fix | functional result | regression |
|---|---|---|
| #3 null-byte / binary bind params | binary int+bytea params now ACCEPTED (were rejected) | none |
| #1 Postgres trust wiring | password-less trust connect now SUCCEEDS (was rejected) | none |
| #2 COPY FROM STDIN dispatch | COPY FROM STDIN now WORKS (was a parser error) | none |
| **cumulative** | all three blockers cleared | **1.003× — no regression** |

A separate Full bug surfaced while benchmarking — rapid **psycopg3** (extended/
pipelined) SELECTs desync (`D` without `T`); psycopg2 (simple protocol) is
unaffected. Tracked as a follow-up, not part of these three fixes.
